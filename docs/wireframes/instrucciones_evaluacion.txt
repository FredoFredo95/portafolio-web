Instrucciones evaluación 3

Descripción General de la Evaluación

La Evaluación N°3 corresponde al desarrollo de un sistema web dinámico denominado:
“Portafolio Web Profesional Autoadministrable”

El proyecto tendrá como finalidad desarrollar una aplicación web moderna, responsive y funcional, que permita presentar información profesional del estudiante mediante un sistema dinámico desarrollado con tecnologías frontend y backend.

El sistema deberá integrar:

    HTML5
    CSS3
    Bootstrap 5.3.8
    JavaScript
    PHP
    MySQL
    AJAX (si aplica)
    JSON (si aplica)
    GitHub
    Herramientas de Inteligencia Artificial como apoyo al desarrollo

Objetivo de la Evaluación

Diseñar e implementar un portafolio web profesional autoadministrable, aplicando buenas prácticas de desarrollo web, diseño responsive, integración frontend-backend, organización estructural y uso ético de herramientas de Inteligencia Artificial.


Resultado de Aprendizaje Asociado
RA2

“Desarrolla aplicaciones web socialmente responsables para un sector productivo y/o social de un contexto real, utilizando herramientas de programación considerando los requerimientos del cliente, desde su fundamentación, planificación, diseño, escritura, resguardando estándares de calidad y normativa vigente”.
Descripción General del Proyecto

El estudiante deberá desarrollar un portafolio web dinámico y administrable que permita visualizar:

    Información personal
    Habilidades técnicas
    Tecnologías dominadas
    Proyectos realizados
    Formulario de contacto
    Sistema de autenticación
    Dashboard administrativo

El sistema debe estar inspirado y desarrollado en base al wireframe previamente diseñado y entregado por cada estudiante.


Requerimientos Obligatorios del Sistema
1. Navbar Responsive

Debe incluir:

    Nombre o logo personal
    Enlaces de navegación:
    Biografía
    Habilidades
    Tecnologías
    Proyectos
    Contacto
    Botón de inicio de sesión

2. Biografía Profesional

Debe contener:

    Fotografía o avatar
    Nombre completo
    Descripción personal/profesional
    Presentación breve

3. Habilidades y Herramientas

Debe mostrar:

    Tecnologías dominadas
    Íconos o logos
    Distribución organizada mediante Bootstrap

Ejemplos:

    HTML
    CSS
    JavaScript
    PHP
    MySQL
    Bootstrap
    GitHub
    IA aplicada al desarrollo web

4. Tecnologías Dominadas

Puede representarse mediante:

    Barras de progreso
    Tarjetas
    Etiquetas
    Indicadores visuales

5. Proyectos Realizados

Cada proyecto deberá incluir:

    Título
    Descripción
    Imagen o captura
    Enlace demo
    Enlace GitHub

6. Formulario de Contacto

Debe contener:

    Nombre
    Correo
    Asunto
    Mensaje
    Botón de envío

7. Sistema de Inicio de Sesión

Debe considerar:

    Login funcional
    Validación de acceso
    Protección de acceso administrativo

8. Dashboard Administrativo

Debe permitir administrar:

    Biografía
    Habilidades
    Tecnologías
    Proyectos

El dashboard puede incluir:

    CRUD
    Formularios dinámicos
    Gestión de contenido

Requerimientos Técnicos Obligatorios
Frontend

    HTML5 estructurado
    CSS3 organizado
    Bootstrap 5.3.8
    Responsive Design
    JavaScript funcional

Backend

    PHP
    MySQL
    Conexión a base de datos
    Validación de formularios

AJAX y JSON

Opcional según complejidad del proyecto, pero evaluable positivamente si se implementa correctamente.


Uso Obligatorio de GitHub

Cada estudiante deberá:

    Mantener repositorio GitHub actualizado.
    Realizar commits periódicos durante el desarrollo.

Criterio esperado

    5 commits o más → máximo puntaje.
    Menos commits → menor evidencia de desarrollo progresivo.

Uso Obligatorio de Inteligencia Artificial

El estudiante deberá documentar:
1. Herramientas IA utilizadas
Ejemplos:

    ChatGPT
    Claude
    Gamma
    Perplexity
    GitHub Copilot

2. Prompts utilizados

Debe incluir ejemplos reales.


3. Resultados generados

Código, ideas, estructuras o soluciones obtenidas.


4. Ajustes realizados

Correcciones o modificaciones efectuadas por el estudiante.


5. Reflexión crítica

Análisis sobre:

    utilidad,
    ventajas,
    limitaciones,
    aprendizaje obtenido.

Entregables Obligatorios

El estudiante deberá entregar:

✅ Proyecto completo en Producción (https://teclab.uct.cl/~usuario/)

✅ Scripts para generar Base de datos SQL en archivo bd.sql

✅ Repositorio GitHub

(Atención: el archivo README.md debe incluir el enlace al proyecto en producción)

✅ Documento de uso de IA

✅ Evidencia del wireframe

✅ Archivo o enlace del diseño Figma utilizado como base del desarrollo

(El diseño Figma debe incorporarse también dentro del repositorio GitHub)

✅ Capturas o video breve funcional (opcional recomendado)


Consideraciones Importantes
Fidelidad respecto al wireframe

El proyecto final deberá mantener coherencia visual y estructural respecto al wireframe previamente desarrollado y entregado.

Se evaluará:

    similitud estructural,
    organización visual,
    navegación,
    distribución de contenidos.

Además, durante la presentación final se contrastará el desarrollo implementado versus el diseño presentado originalmente en Figma.


Ponderación de la Evaluación

La evaluación estará dividida en dos componentes:
1. Desarrollo Práctico del Proyecto — 60%

Corresponde al desarrollo completo del sistema web y a todos los entregables solicitados en esta evaluación.

Este componente considera:

    funcionamiento general del sistema,
    estructura del proyecto,
    calidad del desarrollo frontend y backend,
    uso de GitHub,
    implementación técnica,
    integración de base de datos,
    evidencia del uso de IA,
    coherencia con el wireframe y diseño Figma,
    despliegue en producción.

2. Presentación y Defensa Técnica — 40%

La presentación se realizará el día lunes 25 de mayo a contar de las 11:30 hrs.

Durante esta instancia, cada estudiante deberá:

    Presentar su proyecto funcionando en producción desde el servidor TECLA habilitado.
    Mostrar el repositorio GitHub completo.
    Explicar la estructura de carpetas y organización del código.
    Responder preguntas técnicas realizadas en vivo sobre el desarrollo implementado.
    Mostrar el diseño realizado en Figma para contrastar el resultado final versus la propuesta original de diseño.

En esta etapa se evaluará especialmente:

    dominio técnico del proyecto,
    comprensión del código desarrollado,
    capacidad de explicar decisiones técnicas,
    coherencia entre diseño y desarrollo,
    presentación profesional del trabajo realizado.

Entrega Oportuna
Entrega máxima (dentro de plazo)

23:59 hrs del 27 de mayo.

La entrega fuera de plazo afectará el criterio correspondiente en rúbrica.


Cierre definitivo de entregas

11:30 hrs del 28 de mayo.

Posterior a esa hora no se recibirán entregas adicionales.


Recomendaciones Finales para el Desarrollo

Se recomienda:

    Mantener estructura limpia y ordenada.
    Utilizar Bootstrap de forma consistente.
    Diseñar pensando en experiencia de usuario.
    Aplicar buenas prácticas frontend y backend.
    Utilizar GitHub desde el inicio del proyecto.
    Probar el sistema continuamente.
    Documentar correctamente el uso de IA.
    Mantener coherencia visual con el wireframe original.
    Preparar anticipadamente la presentación y defensa técnica del proyecto.


Reflexión Final
El desarrollo de soluciones web modernas requiere integrar diseño visual, programación frontend y backend, manejo de datos, control de versiones, pensamiento crítico y uso responsable de Inteligencia Artificial para resolver problemas reales de manera profesional y ética.
